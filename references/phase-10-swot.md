# Phase 10: SWOT聚合

## 执行方式

不搜索。纯收敛工作。把前8个Phase的所有发现汇总到一个决策框架里。

## 聚合规则

- 每个SWOT象限至少填2条，最多5条
- 每条必须标注证据来源（来自哪个Phase的哪条数据）
- 优势和劣势是相对于"我们"的（如果用户没提供自己站背景，标注"通用视角"）
- 机会必须是"市场给了但竞对没抓住的"（不是"我们可以做的"——那是行动计划）
- 威胁是"竞对对我们构成的"（不是品类通用风险——那另说）

## 输出格式

```
竞对 SWOT (相对于我们):

S 优势（他们做得好的 — 我们要学/避）:
  1. [优势] — 证据: [Phase X数据]
  2. ...

W 劣势（他们做不好或没做的 — 我们的突破口）:
  1. [劣势] — 证据: [Phase X数据]
  2. ...

O 机会（市场给了但竞对没抓住的）:
  1. [机会] — 为什么竞对没抓住？
  2. ...

T 威胁（他们对我们构成的）:
  1. [威胁]
  2. ...
```

## 策略分类

基于SWOT，把后续行动分为三类：

```
✅ 可抄策略（立即）:
  → 竞对已验证的高ROI动作，我们直接复用
  → 判断标准: 投入低、确定性高、不需要创新

🥊 可打策略（立即）:
  → 在竞对弱项上建立我们的差异化
  → 判断标准: 竞对弱且我们能做好的

⏳ 可建壁垒（中长期）:
  → 竞对抄不走的：内容资产/社区/品牌/IP
  → 判断标准: 投入大但长期复利高
```

## 注意事项

- 不要为了凑数而填——宁可空缺标注"此处无重大发现"
- 差距过大的领域不要硬写"可打"——标注"当前不可行"
- 如果竞对太弱（比如运营基本功都不过关），优势栏可能确实很少——这是真实发现
