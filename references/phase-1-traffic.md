# Phase 1: 流量底盘摸底

## 数据采集清单

逐项执行以下检查，标注每项的数据来源和可信度（确认/估算/推断）：

### 月访问量
```
搜索: "[域名] traffic similarweb monthly visits"
搜索: "[域名] website traffic rank"

判断标准:
- 有公开数据 → 标注"确认"
- 仅有排名（如#1.6M）→ 用排名反推流量，标注"估算"
- 完全无数据 → 标注"推断"，基于Trustpilot评论量/社媒存在感等信号估算
```

### 增长趋势
```
搜索: "[域名] traffic growth trend MoM"
备选: SimilarWeb 趋势图（如果有的话）

判断: 增长 / 平稳 / 衰退
```

### 设备分布
```
移动端占比 > 60% → 移动优先
移动端 40-60% → 并重
移动端 < 40% → 桌面为主（高客单价/B2B品类常见）
```

### 新访 vs 回访
```
新访 > 70% → 靠获取新流量驱动（广告/SEO/社媒）
回访 > 40% → 品牌/社区/订阅驱动
```

### 访问质量三角
- 平均停留时长: <1min短 / 1-3min中 / >3min长
- 页均浏览数: <2低 / 2-4中 / >4高
- 跳出率: <40%低 / 40-60%中 / >60%高

三高（高停留+高页数+低跳出）= 内容/产品匹配好
高流量+高跳出 = 流量质量差或着陆页不行

### 流量来源国家 Top 5
确认主战场和次战场。如果有多语言站点（如/fr/ /ko/），匹配国家分布。

### 域名信息
- 域名年龄: WHOIS查询
- 建站平台: BuiltWith / Wappalyzer / 页面特征

## 输出

**关键假设（一句话）**：
```
"_____端是主战场 | 站是_____（入口/终点） | 核心成交发生在_____"
```

**一句话定性**：这个站是什么类型的玩家？

## 数据不足时的处理

小站点（全球排名>1M）常见SimilarWeb无数据。此时：
1. 用Trustpilot评论量/频率反推订单量
2. 用社媒存在感推断品牌阶段
3. 用Google site:搜索结果推断内容体量
4. 明确标注所有推断值为"估算"
