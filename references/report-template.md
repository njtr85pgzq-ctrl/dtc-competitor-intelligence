# 竞调报告输出模板

## 使用方法

执行完 Phase 1-11 后，按此模板生成最终报告文件。

## 文件路径

`~/dtc-reports/[竞对名称]-竞对拆解-[日期].md`（目录不存在则自动创建）

## 模板结构

```markdown
# [竞对名称] 竞对深度拆解报告

**调研日期**: [YYYY-MM-DD]
**竞对网址**: [url]
**调研方法**: DTC竞对独立站深度拆解框架 v3（集成 marketing-skills 172 skill）
**数据来源**: [列出所有实际使用的数据源]

⚠️ **数据质量声明**: [如果数据不足，在此声明]

---

## Executive Summary（30秒速读）

> 放在报告最前面。任何老板/投资人/团队都能在30秒内看懂。

```
[竞对名称] | 年GMV ~$X | [核心获客模式: 如"SEO+邮件驱动"]
一句话: [最重要的战略判断]
Top 3 立刻做的事:
  1. [动作] — [抄/打] — [预期效果]
  2. [动作] — [抄/打] — [预期效果]
  3. [动作] — [抄/打] — [预期效果]
```

示例:
```
DME Superstore | 年GMV ~$2M | SEO+邮件自动化驱动
一句话: 运营跟不上增速的初创DTC，我们同价+好服务就能赢
Top 3 立刻做的事:
  1. Klaviyo邮件自动化流 — 抄 — 预期提升转化20%+
  2. 免费退货政策 — 打 — 直接吃他们差评流量
  3. Google Shopping广告 — 打 — 竞对完全没防御

## 注意

- 每个Phase都要有数据标注：确认 / 估算 / 推断
- 小站点数据不足是正常的，不要为了凑数据而编造
- 报告遵循"功利性"原则：每条发现都要回答"所以呢？对我有什么用？"
- **Executive Summary 必须在报告最前面**，30秒内可读完
