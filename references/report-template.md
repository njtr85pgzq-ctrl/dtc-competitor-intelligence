# 竞调报告输出模板

## 使用方法

执行完Phase 1-10后，按此模板生成最终报告文件。

## 文件路径

`d:/projects/dtc-competitor-intelligence/examples/[竞对名称]-竞对拆解-[日期].md`

## 模板结构

```markdown
# [竞对名称] 竞对深度拆解报告

**调研日期**: [YYYY-MM-DD]
**竞对网址**: [url]
**调研方法**: DTC竞对独立站深度拆解框架 v2
**数据来源**: [列出所有实际使用的数据源]

⚠️ **数据质量声明**: [如果数据不足，在此声明]

---

| 哪些做法我们能照抄？ | |
```

## 注意

- 每个Phase都要有数据标注：确认 / 估算 / 推断
- 小站点数据不足是正常的，不要为了凑数据而编造
- 报告遵循"功利性"原则：每条发现都要回答"所以呢？对我有什么用？"
