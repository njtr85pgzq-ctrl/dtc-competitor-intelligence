# Phase 6: 转化漏斗与GMV测算

## 漏斗拆解

```
流量 → [着陆] → [浏览] → [加购] → [结账] → [支付] → GMV
       ↓         ↓         ↓        ↓        ↓
     跳出率      PDP转化   加购率   结账率   支付成功率
```

每个环节对标行业benchmark，找到竞对（和我们）的漏斗断裂点。

## GMV三角验证法

用三种方法交叉验证，取交集得到合理区间：

### 方法一：流量反推
```
月流量 × 估算转化率 × 估算客单价 = GMV
数据来源: SimilarWeb流量 + 行业转化率benchmark + 站内商品价格

行业转化率benchmark:
- DTC一般: 1-3%
- 高客单价(>$500): 0.5-1%
- 低客单价(<$50): 2-5%
- 配件/替换件: 1-2%
```

### 方法二：评论数反推
```
Trustpilot/Judge.me总评论数 ÷ 站点月数 = 月均评论数
月均评论数 ÷ 行业留评率(1-5%) = 月均订单数
月均订单数 × 估算AOV = 月GMV
```

### 方法三：社媒互动反推
```
社媒粉丝 × 互动率 → 活跃用户池 → 购买转化 → GMV
适用于社媒活跃的品牌，社媒弱的跳过此方法
```

### 取值
取三种方法的交集区间。如果某方法推算结果与其他方法差异>50%，标注为离群值不采用。

## 多渠道收入估算

如果竞对是多渠道运营（如Amazon+独立站+Alibaba），分别估算每个渠道的GMV。

标注主力利润来源（通常B2B做现金流，B2C做品牌）。
